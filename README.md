# Reflection

This project helped me better understand how to write code that is maintainable, readable, and adaptable. One of the best examples of this was the CRUD Python module I created in Project One. By keeping the database connection and database operations in a separate module, I did not have to rewrite that same logic again inside the dashboard code for Project Two. This made the dashboard easier to read and easier to manage. It also made the project more adaptable because if I ever need to change the database connection, update a query, or reuse the same database functions, I can do that in one place. In the future, I could use this same CRUD module in other Python projects that need to connect to MongoDB, such as another dashboard, a reporting tool, or a different web application.

When I approach a problem as a computer scientist, I try to break it into smaller parts and solve each part step by step. That is how I approached this project. First, I needed to understand the client’s requirements. Then I focused on building the database connection, testing the CRUD module, and finally connecting that work to the dashboard. This felt different from some of my earlier assignments because this project required me to think about the full flow of the system instead of just one isolated piece. I had to think about the database, the code, the dashboard layout, and the user experience all together. In the future, when creating databases for other client requests, I would continue using this same approach by starting with the client’s needs, designing the database around those needs, and then building tools that make the data easier to use.

Computer scientists do more than just write code. They solve problems by building systems that help people work more efficiently and make better decisions. That matters because many companies depend on technology to organize information and support daily work. In this project, my work could help a company like Grazioso Salvare by turning a large amount of shelter data into something much easier to understand and use. Instead of searching through raw records by hand, users can filter the data, view results in a table, and use charts and maps to better understand the information. That saves time, reduces mistakes, and helps the company make better decisions about which animals may be strong candidates for rescue training.
